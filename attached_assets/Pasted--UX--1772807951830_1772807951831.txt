═══════════════════════════════════════
ЗАДАЧА 1 — ПЕРЕДЕЛАТЬ ГЛАВНУЮ СТРАНИЦУ
═══════════════════════════════════════

ТЕКУЩАЯ ПРОБЛЕМА:
Сейчас на главной странице отображаются карточки БЛЮД, и под каждым блюдом написано имя повара. Это неправильный UX.

НУЖНЫЙ UX — 2 уровня навигации:

── УРОВЕНЬ 1 — ГЛАВНАЯ СТРАНИЦА (список поваров/кухонь) ──

На главной странице показывать карточки ПОВАРОВ (кухонь), а не блюд.

Каждая карточка повара должна содержать:
- Фото повара / аватар кухни
- Имя повара (first_name + last_name)
- Тип кухни (French, Italian, Georgian, Russian и т.д.)
- Рейтинг повара (звёзды + число)
- Количество блюд в меню (например "7 блюд")
- Превью 2-3 блюд в виде маленьких картинок (горизонтальный ряд)
- Вся карточка кликабельна → переход на страницу повара

Данные: таблица users (role='cook') + cook_profiles + dishes (группировать по cookId)

── УРОВЕНЬ 2 — СТРАНИЦА ПОВАРА /cook/:cookId ──

При клике на карточку повара — переход на страницу конкретного повара.

На странице повара показывать:
- Шапка: фото, имя, рейтинг, тип кухни, описание (bio)
- Полное меню: все блюда этого повара в виде сетки карточек
- Карточка блюда: фото, название, цена, время приготовления, кнопка "В корзину"
- Фильтрация блюд по категории (если у повара несколько категорий)

Технические требования:
1. Создать новый компонент CookCard.tsx для карточки повара
2. Создать новую страницу CookPage.tsx
3. Добавить роут /cook/:cookId в react-router
4. API endpoint: GET /api/cooks — список поваров с количеством блюд и превью
5. API endpoint: GET /api/cooks/:cookId/dishes — все блюда конкретного повара
6. Сохранить существующие фильтры (категории, тип кухни) — перенести на главную для фильтрации поваров
7. Поддержать оба языка (RU/EN) через i18n
